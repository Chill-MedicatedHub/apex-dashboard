# Apex Sales Dashboard — Legacy Operations

A self-contained dashboard for the Apex Trading "All Product Sales" report.
The scraper pulls data via Apex's internal API using your browser session;
the dashboard reads the result and renders KPIs, charts, and tables.

```
apex-dashboard/
├── scraper.py          ← pulls data from Apex
├── refresh_loop.py     ← optional: pulls every 15 min
├── dashboard.html      ← the dashboard UI
├── sales_data.json     ← generated by scraper (don't edit)
├── .env                ← your secrets (create from .env.example)
├── run.sh              ← one-click runner (macOS/Linux)
├── run.bat             ← one-click runner (Windows)
└── requirements.txt
```

---

## 1. First-time setup (5 minutes)

### A. Install Python dependencies
```bash
pip install -r requirements.txt
```

### B. Create your `.env` file
Copy the template:
```bash
# macOS / Linux
cp .env.example .env

# Windows
copy .env.example .env
```

### C. Grab your session cookie from Apex

This is the only "secret" the scraper needs. You'll redo this every few weeks
when the cookie expires.

1. Log into Apex in Chrome at https://app.apextrading.com
2. Open the **All Product Sales** report
3. Press **F12** → click the **Network** tab → click **Fetch/XHR**
4. Click the **Run** button on the report
5. In the network list, click **`run-all-product-sales-report`**
6. On the right, click **Headers** → scroll to **Request Headers**
7. Find the **`Cookie:`** row and **copy the entire value** (it's long — starts with stuff like `_gid=...; _fbp=...; sa-user-id=...` and continues)
8. Open `.env` in a text editor and paste it between the quotes:
   ```
   APEX_COOKIE="_gid=GA1...; _fbp=fb...; sa-user-id=...; (full string)"
   ```
9. Save the file. **Do not share this file or commit it to git.**

---

## 2. Run it

### Easy mode — one command:
```bash
# macOS / Linux
./run.sh

# Windows — double-click run.bat
```

This pulls fresh data, starts a local web server, and opens the dashboard.

### Manual mode:
```bash
# 1. Pull data
python scraper.py

# 2. Start the server
python -m http.server 8000

# 3. Open http://localhost:8000/dashboard.html in your browser
```

---

## 3. Keeping data fresh

### Option A — Background refresh loop (simplest)
Open a second terminal alongside the dashboard and run:
```bash
python refresh_loop.py
```
This pulls every 15 minutes. The dashboard auto-refreshes every 60 seconds
and will pick up the new data automatically.

### Option B — Cron (macOS/Linux)
```bash
crontab -e
```
Add this line to pull every 15 minutes:
```
*/15 * * * * cd /full/path/to/apex-dashboard && /usr/bin/python3 scraper.py >> scraper.log 2>&1
```

### Option C — Task Scheduler (Windows)
1. Open **Task Scheduler** → **Create Basic Task**
2. Trigger: **Daily**, repeat every **15 minutes**
3. Action: **Start a program**
   - Program: `python`
   - Arguments: `scraper.py`
   - Start in: `C:\path\to\apex-dashboard`

---

## 4. When the cookie expires

You'll see this in the terminal:
```
ERROR: Apex rejected the request (status 401).
Your session cookie has likely expired.
```

Fix: repeat **Step 1.C** above with a fresh cookie value. Takes 30 seconds.

---

## 5. Customizing

- **Pull more history** → edit `APEX_DAYS_BACK` in `.env` (default 90 days)
- **Pull more rows** → edit `APEX_ROW_LIMIT` in `.env` (default 5000)
- **Filter by brand/rep/etc.** → use the dashboard filters; the scraper pulls everything and the dashboard filters client-side

## Troubleshooting

**"No data yet" in the dashboard**
The scraper hasn't run successfully yet, or `sales_data.json` is missing.
Run `python scraper.py` and check for errors.

**"Could not load sales_data.json" / CORS error**
You're opening `dashboard.html` directly via `file://`. Use `./run.sh`
(or `python -m http.server 8000`) so it loads via `http://localhost:8000`.

**Scraper returns 0 rows**
Check your date range (`APEX_DAYS_BACK`) and confirm you actually have sales
in that window. Try widening to 365.

**Scraper returns 401/403**
Cookie expired — see step 4.

## Security notes

- Your `.env` contains a live session for your Apex account. Treat it like a password.
- `.gitignore` excludes `.env` and `sales_data.json` from git by default.
- The dashboard runs entirely on `localhost`. Nothing leaves your machine.
- Use a dedicated Apex user account for this if you want to limit blast radius.
